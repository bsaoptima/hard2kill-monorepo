## SLIDESHOW FORMATS

### Format 1: Single Slide Puncher
- 1 slide only
- Grid background + glass card
- Ultra bold claim
- Screenshot and post
- Example: "$847 this week / Playing geography"

### Format 2: 3-Slide Story
- Slide 1: Problem/waste
- Slide 2: Discovery
- Slide 3: Result
- Example:
  - S1: "Wasted 1000 hours on ranked / Made $0"
  - S2: "Found out you can play geography for money"
  - S3: "$500 in 3 weeks / Same skills, different game"

### Format 3: Contrast Pair
- Slide 1: Before
- Slide 2: After
- Example:
  - S1: "Your friends / Play for fun"
  - S2: "You / Play for money"

### Format 4: List Stack
- 5 slides, rapid fire
- Each slide = one line
- Example series:
  - "$20 → $40"
  - "$40 → $80"
  - "$80 → $160"
  - "$160 → $320"
  - "Same day"

### Format 5: Brazilian Localized
- Mix Portuguese + English
- Regional flex
- Example:
  - "Brasileiro / Making dollars"
  - "Geography skills / PIX real"

## CONTENT PRINCIPLES

### DO:
- Lead with numbers ($ amounts, time, percentages)
- Use contrast (before/after, them/you, wrong/right)
- Be polarizing (strong claims trigger reactions)
- Keep it short (5-10 words per line MAX)
- Use "you" language (direct address)
- Reference cultural touchpoints (gaming, ranked, grinding)

### DON'T:
- Use gambling vocabulary ("bet", "wager", "casino", "aposta")
- Explain too much (kill the curiosity)
- Be humble (cocky performs better)
- Add logos or branding (feels like an ad, gets scrolled)
- Use more than 2 lines per slide

## WHAT TO DELETE FROM OLD LIBRARY

The existing content library (GEOSTAKES_CONTENT_LIBRARY.md) is video-focused with long captions and DM flows. That's fine for video content but NOT for slideshow format.

**Delete/Ignore for slideshow content:**
- Long captions (nobody reads them on slides)
- DM automation scripts (different strategy)
- Video production notes
- Multi-paragraph storytelling

**Keep for video content:**
- Educational formats (tutorials work as videos)
- Screen recording concepts
- Match replays with commentary

**Recommendation:** Keep both libraries separate:
- GEOSTAKES_CONTENT_LIBRARY.md = video content
- SLIDESHOW_HOOKS.md = this file = slide content

Don't merge. Different formats, different execution.

## METRICS TO TRACK

For each slide series, track:

| Metric | Target | Why It Matters |
|---|---|---|
| **Saves** | >5% | Indicates high value, people want to reference it later |
| **Shares** | >3% | Viral coefficient, free distribution |
| **Completion rate** | >70% | For multi-slide, did they scroll through all? |
| **Comments** | >100 per 10K views | Engagement signal for algorithm |
| **Profile visits** | >8% | Interest in source |
| **Link clicks** | >2% | Intent to try |

If a slide format hits these targets, make 10 more variations of it.

If it doesn't, kill it after 3 attempts.

No attachment to creative. Only data matters.
